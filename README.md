# Motheretreat — Автоматизация заявок

Автоматизация обработки лид-заявок для агентства послеродового сопровождения **Motheretreat** (Майами, США).

Будущая мама заполняет форму → заявка автоматически попадает в Google-таблицу → команда мгновенно получает уведомление в Slack. Заявки со статусом **«Срочный!»** приходят с особым приоритетом — пингом всего канала (`@channel`).

> **Тестовое задание.** Реализовано в **двух вариантах**: основной на **Zapier** (как требует ТЗ) и бонусный на **Google Apps Script** (чистый код, без подписок и задержек).

---

## 🎯 Задание

> Будущая мама заполняет форму на сайте агентства. Поля: **Имя, Телефон для связи, Статус запроса** (выпадающий список: Обычный / Срочный!). Нужно автоматически заносить данные в общую таблицу и отправлять уведомление команде в Slack. Если запрос «Срочный» — уведомление приходит с особым приоритетом.
> Стек: **Google Forms + Google Sheets + Zapier**.

---

## 📦 Две реализации

| | Вариант 1 — Zapier *(по ТЗ)* | Вариант 2 — Apps Script *(бонус)* |
|---|---|---|
| Стек | Google Forms + Sheets + **Zapier** | Google Forms + Sheets + **Apps Script** |
| Ветвление Срочный/Обычный | Zapier **Paths** | код (`if`) |
| Скорость уведомления | ~1–2 мин (polling) | **мгновенно** |
| Стоимость | Zapier **Pro** (Paths — платная фича) | **бесплатно** |
| Код в репозитории | — *(Zap живёт в аккаунте Zapier)* | [`/apps-script`](apps-script/) |

Обе версии шлют в Slack одинаковые сообщения:

- **Срочный!** → 🚨 `@channel` + Имя / Телефон / Статус
- **Обычный** → 📝 тихое сообщение без пинга

## 💡 Почему две версии

ТЗ требует Zapier — он собран и работает. Но у Zapier есть нюансы: ветвление (Paths) доступно только на платном тарифе **Pro**, а триггер Google Sheets срабатывает с задержкой ~1–2 минуты (polling).

Поэтому в довесок реализована версия на **Google Apps Script** — тот же сценарий, но **бесплатно, без подписок и мгновенно** (срабатывает прямо на отправку формы). Это показывает, что задача решается не только в no-code конструкторе, но и кодом.
